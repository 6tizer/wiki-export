## 一句话摘要

OpenClaw 这次围绕 Molty SOUL 强化的不是模型或工具，而是 Agent 的人格层，通过直接改写 [SOUL.md](concepts/SOUL.md.md)，让智能体从“企业客服味执行器”走向“敢判断、能长期协作的搭档”。

## 关键洞察

- Molty SOUL 的核心不是“加个风格包”，而是把人格、语气与协作风格当成可配置系统层

- 对长期使用来说，决定体验上限的往往不是功能够不够，而是 Agent 有没有判断感和人味

- 所谓“更辣”的本质，是鼓励 Agent 在必要时进行 [Push back 协作](concepts/Push back 协作.md)，指出用户方案中的明显问题

- 这类改造应只动语气层，不应破坏业务规则、安全边界与审批纪律

- 从产品演进看，OpenClaw 正把竞争点从模型层、工具层、记忆层，进一步推向 [Agent 人格层](concepts/Agent 人格层.md)
