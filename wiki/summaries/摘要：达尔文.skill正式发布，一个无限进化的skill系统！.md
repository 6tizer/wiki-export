## 一句话摘要

**达尔文.skill** 把 Karpathy 的 autoresearch 思路迁移到 Claude Code 的 skill 优化场景，用“评估—改进—测试—保留或回滚”的棘轮循环，让大量 skill 能持续进化而不积累退步。

## 关键洞察

- 它解决的不是“如何从零写一个新 skill”，而是“当你已经有几十个 skill 时，如何系统性提升质量”

- 核心机制是棘轮式优化，只有分数提升的修改会被保留，退步修改自动回滚

- 相比 autoresearch 的全自动实验，它额外加入了 **Human in the Loop**，把主观判断留给人类做终审

- 它把结构规范和真实效果拆成双重评估，并用独立子 Agent 打分，减少“自己改自己评”的偏差

- 它与 [女娲.skill](concepts/女娲.skill.md) 形成“生成 skill + 优化 skill”的闭环，指向一套可持续演化的 skill 基础设施
