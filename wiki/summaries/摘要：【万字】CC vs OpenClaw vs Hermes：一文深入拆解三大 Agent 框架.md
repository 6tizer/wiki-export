## 一句话摘要

这篇文章把 Claude Code、OpenClaw 与 Hermes 放回三条不同的 Agent 路线里比较：前者是“养人”的副驾驶，后者分别是“养社区”的技能生态与“养自己”的自进化 Agent，框架之争本质上是在押注不同的人机关系与能力沉淀方式。

## 关键洞察

- Claude Code 的核心不是更强的自主性，而是把 AI 定位为由人主导的副驾驶；定价、配置层级与 task budget 都围绕“人监督 AI”展开。

- OpenClaw 的真正壁垒不只是框架本身，而是以 ClawHub 为中心的社区技能广场；能力扩张速度快，但安全面也随生态一同膨胀。

- Hermes 试图把 Agent 从一次性工具变成会持续积累经验的“云端员工”，通过自我评估、技能沉淀与迁移工具放大长期复利。

- 三者在“记忆”上的定义完全不同：Claude Code 偏规则手册，OpenClaw 偏身份 + 技能拼装，Hermes 偏分层记忆与长期检索，因此决定了用户与 Agent 的关系深浅。

- 对个人或团队而言，选型不只是比较功能，而是在选择未来把知识、工作流与习惯沉淀到哪里；频繁横跳会不断清空 Agent 的复利。
