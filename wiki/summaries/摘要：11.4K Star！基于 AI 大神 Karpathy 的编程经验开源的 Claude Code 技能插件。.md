## 一句话摘要

开源插件 andrej-karpathy-skills 将 Andrej Karpathy 对 LLM 编程陷阱的四条核心原则封装为 Claude Code 的 [CLAUDE.md](http://claude.md/) 配置，强制 AI 思考先行、简洁至上、精准改动、目标驱动。

## 关键洞察

- **LLM 编程三大痛点**：盲目假设不提问、过度设计代码臃肿、副作用大乱改无关代码——这些问题 Karpathy 早已精准总结

- **四条铁律**：Think Before Coding（提问代替猜测）、Simplicity First（最少代码）、Surgical Changes（精准改动，每行可追溯需求）、Goal-Driven Execution（给成功标准而非步骤清单）

- **两种接入方式**：全局插件安装（`/plugin install`）或项目级 [CLAUDE.md](http://claude.md/) 文件，均无需修改代码，门槛极低

- **可验证的改进信号**：diff 更干净、减少返工重写、AI 会在实现前先提出澄清问题

- **设计哲学**：「不要告诉它做什么，给它成功标准，然后看着它去做。」——Karpathy

## 个人评注

这个插件与 Tizer 的工作流高度相关：

- **Coding Agent / OpenClaw**：OpenClaw 内部如果用到 Claude Code 作为子 Agent 执行编码任务，接入 andrej-karpathy-skills 可以显著减少 AI 的过度设计和乱改代码问题，降低 HITL 干预频率

- **HITL 流程**：「精准改动」原则要求每行改动都可追溯到用户需求，这与 HITL 审核逻辑天然吻合——减少噪声 diff 就是减少审核负担

- **知识管理**：[CLAUDE.md](http://claude.md/) 本身是一种「可执行的知识沉淀」模式，可以把团队最佳实践直接注入 AI 行为，值得在 Tizer 的内容 Pipeline 配置中推广
