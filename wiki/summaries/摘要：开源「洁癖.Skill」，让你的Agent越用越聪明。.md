## 一句话摘要

洁癖.Skill 是一个跨平台开源 Agent Skill，通过在每次任务结束后自动审查并同步项目的三层知识体系（Agent 记忆、[CLAUDE.md](http://claude.md/)、docs/），解决 AI 协作开发中「文档越来越烂、Agent 越用越笨」的根源问题。

## 关键洞察

- **Agent 变笨的根源不是模型能力不足，而是上下文（文档+记忆）混乱**：过期信息比没有信息更糟糕，因为 Agent 会基于错误前提做决策，而不是承认不知道

- **AutoDream 只管记忆层，不管文档层**：Claude Code 的 Auto Dream 功能虽然能整理记忆文件，但不会触碰项目根 [CLAUDE.md](http://claude.md/) 和 docs/ 目录，导致三层知识中有两层长期失修

- **三层知识服务不同受众，不可互替**：Agent 记忆（给 Agent 自己）、[CLAUDE.md](http://claude.md/)（给项目内 AI）、docs/（给其他人/下游系统），[CLAUDE.md](http://claude.md/) 写了「新增五个路由」不等于 docs 里写了「下游怎么接这五个路由」

- **核心原则是「合并优于追加，删除优于保留」**：与大多数人直觉相反，信息准才是优势，不是信息多

- **变更影响矩阵实现精准定位**：不仅检查本次对话的新事实波及哪些文档，还检查跨项目依赖——改了项目 A，依赖它的项目 B 的文档也要同步

## 个人评注

洁癖.Skill 的设计理念与 Tizer 的知识管理工作流高度契合——本质上都是在解决「知识体系如何跟上变更节奏」的问题。文章提出的「合并优于追加、删除优于保留」原则，与 Wiki Compiler 的去重和归一化逻辑一脉相承。变更影响矩阵的跨项目依赖检查思路，可以借鉴到 OpenClaw 多模块协作场景中——当一个 Skill 更新后，依赖它的其他 Skill 的文档是否也需要同步更新？这是当前 content pipeline 尚未覆盖的维护盲区。
