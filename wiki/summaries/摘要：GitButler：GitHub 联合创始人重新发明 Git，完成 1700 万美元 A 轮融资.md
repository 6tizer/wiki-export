## 一句话摘要

GitButler 试图用虚拟分支和堆叠分支重做 Git 日常工作流，让多任务并行开发和 AI 编程协作不再依赖痛苦的 stash 与切分支操作。

## 关键洞察

- 它重构的是工作流抽象，而不是传统 Git GUI 的皮肤层。

- 虚拟分支非常适合承接多个 Agent 并行产出的改动整理。

- 堆叠分支把大改动拆成可审查的小块，更适合 AI 生成代码的增量合并。

- AI 编程时代，版本控制工具正在从“存代码”转向“管理并行上下文”。
