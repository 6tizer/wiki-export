## 一句话摘要

这条推文宣布一个面向 Andrej Karpathy 的人格资料包已经上线：项目通过整理其公开语料，生成 [SOUL.md](http://soul.md/)、[STYLE.md](http://style.md/)、[MEMORY.md](http://memory.md/) 等文件，以便 AI 代理更稳定地复现其表达风格、偏好边界与思考方式。

## 关键洞察

- 这不是普通的链接汇总，而是对个人公开内容进行结构化蒸馏，目标是生成可直接给 Agent 使用的人格配置。

- 公开材料覆盖 13 篇博客、12 份 YouTube 转录、8 个仓库 README 和 200 条推文，说明该 persona build 依赖高密度、长期、跨媒介语料。

- 项目把人格包拆成 [SOUL.md](http://soul.md/)、[STYLE.md](http://style.md/)、[MEMORY.md](http://memory.md/) 三类文件，分别承载身份特征、表达风格与持续记忆。

- 除静态资料外，条目还编码了 5 种模式与 8 组 tensions / boundaries / pet peeves，说明其目标是让 Agent 在不同语境下保持更细腻的一致性。

## 个人评注

这类“公开语料 → 人格文件”的做法，对 Tizer 当前的 Wiki 编译流很有启发：一方面它说明 summary/concept 之外，还可以把长期输出稳定的人物沉淀为可复用的 entity 档案；另一方面，SOUL / STYLE / MEMORY 这种分层方式也适合映射到 OpenClaw 与 HITL 场景中的身份、表达和记忆管理。
