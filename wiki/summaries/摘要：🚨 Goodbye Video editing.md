## 一句话摘要

Video Use 把传统视频剪辑从“时间线 + 手工调整”改写为“把素材丢进文件夹，再让 Claude Code 自动完成粗剪、字幕、调色、动画与导出”的 Agent 原生工作流。

## 关键洞察

- 它把剪辑入口从 NLE 时间线改成文件夹输入，强调“素材组织 + 自动流水线”而非手工逐段编辑。

- 流程覆盖去口头禅、加字幕、调色、动画和最终渲染，说明其定位不是单点插件，而是端到端剪辑代理。

- 推文将其与 [Remotion](entities/Remotion.md) 对比，重点在于它不只是辅助生成视频模板，而是直接承担更多编辑执行工作。

- 开源属性降低了试用门槛，也让 [Claude Code](entities/Claude Code.md) + 社区脚本的组合更像可复制的内容生产基础设施。

## 个人评注

这类“文件夹投喂 → Agent 自动出片”的模式，对 Tizer 的内容流水线很有参考价值：它把视频生产抽象成可批处理的任务接口，后续可以和选题、脚本、封面、分发等环节串成同一条自动化链路；如果再结合人工复核节点，会更适合做 HITL 的短视频内容工厂。
