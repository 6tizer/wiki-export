## 一句话摘要

Intercom 通过把研发组织切换到以 Claude Code 为核心的 agent-first 开发模式，在 9 个月内实现研发效率翻倍，并把缺陷消化、交付速度、代码质量、稳定性和成本结构一起重做。

## 关键洞察

- 他们故意把“合并 PR 数 / 全体 R&D 人数”设为核心指标，用高压指标暴露组织瓶颈，再围绕瓶颈重构流程。

- Agent-first 不只是让 AI 写代码，而是把技术工作、PR 创建、PR 审批、插件分发和组织协作都迁移到 Agent 驱动模式。

- 交付速度与质量并非零和：缺陷积压下降 54%，创意到上线时间下降 39%，代码变更导致的宕机下降 35%。

- AI 成本快速上升，但按每个 PR 分摊的总成本持续下降，说明工资成本被更高的吞吐量摊薄。

- 真正的瓶颈开始从工具转向人：顶尖 5% 工程师的 PR 吞吐量达到中位数的 6 倍，组织需要把最佳实践产品化并扩散。
