## 一句话摘要

Bloome 把本地 Agent、云端 Agent 和他人 Agent 放进同一个群聊空间，让 Agent 的能力、上下文与协作关系可以像社交关系一样被直接分享和调度。

## 关键洞察

- 它把单个 Agent 的使用方式，从“一个人对一个助手”推进到“多个 Agent 在同一会话里分工协作”。

- 平台自带云端 Agent，可在本地设备离线时承接轻量任务，形成本地与云端互补的执行结构。

- 本地 Agent 的接入门槛被压到接近一键操作，说明 Agent 编排开始从工程玩法走向普通用户可用的产品形态。

- 通过二维码把不同人的 Agent 拉进同一群，意味着 Agent 不再只是个人工具，也开始具备“可分享的协作节点”属性。

- 支持自定义 API 而非强制平台内购，说明产品试图把自己定位为开放式 Agent 协作层，而不是封闭模型入口。

## 个人评注

这篇内容最值得记录的不是某个单一功能，而是一个方向信号：当 Agent 可以被“拉群、转交、共享、@ 出来干活”时，Agent 就开始从个人生产力工具演变成团队协作接口。对 Tizer 的工作流而言，这类产品值得重点关注，因为它和多 Agent 编排、HITL、内容流水线以及未来的 Agent 社交分发场景都有直接关联。
