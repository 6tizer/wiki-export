## 一句话摘要

ScreenKite 把原生 macOS 录屏、快速导出与 AI Agent 驱动的视频后期整合到同一工具里，试图把产品 Demo 和教程视频制作变成更适合 Claude Code、Codex 等编程代理执行的自动化流程。

## 关键洞察

- ScreenKite 的定位不是传统录屏工具，而是“让你喜欢的 AI”直接参与剪视频的 Agent 原生录屏工作台。

- 产品强调原生 macOS、Swift 实现和快速导出，瞄准的是想快速产出 Demo 视频的开发者与产品团队。

- 它把自动转录裁剪、B-roll 生成、动态镜头布局等步骤暴露给 AI 工作流，降低了手动时间轴编辑的比重。

- 配套开源仓库提供 Claude Code skill，说明其生态重点不是单点工具，而是可复用的 AI 视频编辑 pipeline。

- 与 hyperframes 的组合表明，ScreenKite 更像视频编排与落地执行层，而 hyperframes 负责代码驱动的动态视觉生成。

## 个人评注

这类工具对 Tizer 的价值，不只是“更快剪视频”，而是把内容产线进一步推向可编排：前面可以接文章摘录、脚本整理、产品亮点提炼，后面可以接 OpenClaw 或 Coding Agent 去批量生成 demo、教程和宣传素材。对内容工厂来说，真正关键的是把“录屏→转录→补画面→导出”沉淀成可重复调用的 workflow，而不是继续依赖人工逐段拖拽时间轴。
