**一句话摘要**：通过在 OpenClaw 中同时管理 6 个不同人格的 Agent，配合 [SOUL.md](http://soul.md/) 和 [AGENTS.md](http://agents.md/) 通讯录，实现多 Agent 团队协作与任务调度。

**关键洞察**

- 多 Agent 策略：每只 Agent 配置独立人格，避免上下文混乱

- [SOUL.md](http://soul.md/) 文件定义 Agent 个性与工作边界

- [AGENTS.md](http://agents.md/) 通讯录使 Agent 间相互识别，支持跨 Agent 消息路由

- 通过飞书进行 Agent 间通信，形成团队协作闭环

- 重启 gateway 是多 Agent 配置生效的关键步骤

**提取的概念**：[SOUL.md](http://soul.md/) 人格配置、多 Agent 通讯录
