### 一句话摘要

`public-apis` 的真正价值不只是免费接口目录，而是它可以被当作 Agent 的工具地图，让 Agent 在运行时主动发现和组合可用能力。

### 关键洞察

- 把 API 列表喂给 Agent，本质上是在把工具发现从人工配置改成运行时检索。

- `public-apis` 适合做原型期工具探索，但不适合直接当作生产级稳定数据源。

- Agent 能从大规模接口列表里发现人类不容易手动想到的 API 组合。

- 这条路线与 MCP、tool registry 和 function schema 的思路高度一致。
