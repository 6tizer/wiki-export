## 一句话摘要

x402 与 MPP 分别代表了 Agent 支付的两条路线：前者强调开放、逐请求、链上原生，后者强调会话、批量结算和对现有金融网络的兼容。

## 关键洞察

- x402 更像底层协议，适合简单直接的链上微支付场景。

- MPP 用 Session 机制解决高频调用下的支付摩擦，更偏工程实用主义。

- Agent 支付的竞争不只是结算方式之争，本质上也是开放协议与平台分发能力的较量。

- 未来最可能的格局不是单一赢家，而是底层协议、会话层与授权风控层并存。
