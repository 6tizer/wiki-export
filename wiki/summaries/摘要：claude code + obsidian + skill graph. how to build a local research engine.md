## 一句话摘要

这篇文章提出了一套以 Claude Code + Obsidian + 本地 Markdown 图谱为核心的“Research Skill Graph”研究引擎，用结构化方法文件、六镜头分析和可累积知识库，把一次性 AI 检索升级为可复用的本地研究系统。

## 关键洞察

- 文章把研究系统拆成 `index.md`、方法论文件、镜头文件、知识文件、项目目录和来源模板，强调先搭结构，再让 AI 执行研究

- 它提出六镜头研究法，要求 AI 从 technical、economic、historical、geopolitical、contrarian 和 first-principles 六个角度重新思考同一问题，而不是只写一份更长的总结

- 它通过信源分级体系和矛盾处理协议，为研究过程补上证据质量控制与分歧处理机制，减少“AI 套话式研究”

- 它把 Obsidian 的本地 Markdown Vault 与 Claude Code 的读写执行能力结合起来，让研究知识可以持续沉淀、复用和迭代

- 回复线程进一步把 heartbeat 项目与 KAIROS 讨论接进来，说明这套系统可被延展到持续后台运行、记忆整合和主动研究循环的方向

## 个人评注

这类“本地 Markdown + 方法文件 + Agent 执行”的结构，与 Tizer 当前的知识编译链路高度契合。它最大的启发不是某一个工具组合，而是把研究过程拆成可审阅的中间层：方法规则、镜头模板、知识沉淀和矛盾记录都被显式化了。这对后续做 HITL 审核、内容复用、OpenClaw 式持续研究代理，都有很强的可迁移性。
