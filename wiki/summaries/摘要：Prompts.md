## 一句话摘要

这篇 X 书签文章围绕 khazix-skills 仓库，拆解了“横纵分析法”如何从深度研究方法演化为可执行 Skill，并进一步延伸到公众号长文写作的 khazix-writer，展示出一条从研究到写作的 AI 内容生产链路。

## 关键洞察

- [横纵分析法](concepts/横纵分析法.md) 把历时分析与共时比较结合，适合快速理解陌生主题

- [hv-analysis](concepts/hv-analysis.md) 将该方法封装为 Agent Skill，支持并行信息搜集、论文检索、长报告生成与 PDF 输出

- [khazix-writer](concepts/khazix-writer.md) 不只是写作 Prompt，而是一套完整的公众号长文写作方法论，强调五种文章原型、风格约束与“人味”质检

- [反向约束设计](concepts/反向约束设计.md) 与 [四层自检体系](concepts/四层自检体系.md) 是压制 AI 腔、保留作者风格的核心机制

- [khazix-skills](entities/khazix-skills.md) 把个人方法论产品化为可安装、可复用的 Prompt 与 Skill 资产，适合接入 Claude Code、Codex、OpenClaw 等 Agent

## 个人评注

这类“研究 Skill + 写作 Skill”组合与 Tizer 当前的内容编译链路高度契合。前者负责把分散信息结构化，后者负责把结构化研究转成可发布文本，适合作为 OpenClaw / HITL 内容流水线里的中间层资产。若后续把来源引用、关联概念和人工复核节点接起来，就能形成更稳定的知识沉淀闭环。
