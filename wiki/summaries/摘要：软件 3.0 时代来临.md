## 一句话摘要

Karpathy 在 Sequoia AI Ascent 2026 上提出 Software 3.0 范式——用自然语言 prompt 和 Agent 替代传统编码，标志着软件开发从「人写代码」到「人指挥 Agent」的第三次根本性转移。

## 关键洞察

- **三代软件范式演进**：Software 1.0（人写代码）→ Software 2.0（神经网络训练）→ Software 3.0（prompt + context window + Agent + 工具调用），编程界面从代码编辑器变为 context window

- **Vibe Coding 抬高地板，Agentic Engineering 抬高天花板**：Vibe Coding 让非程序员也能构建应用，降低准入门槛；Agentic Engineering 则要求专业开发者用 Agent 系统保证安全性、可靠性和架构品质

- **可验证性决定 AI 进步速度**：AI 在编程、数学等可验证领域进步飞速，但在常识推理等缺乏奖励信号的领域表现参差——这就是「锯齿状智能」（jagged intelligence）

- **Harness 工程成为关键**：Mitchell Hashimoto 提出的 harness 理念——每发现 Agent 犯错就工程化一个永久修复方案——正在成为 Agentic Engineering 的核心方法论

- **LLM 吞噬中间层应用**：MenuGen 等案例表明，某些应用不是被 AI 加速，而是被模型原生能力直接替代，整个中间层软件变得多余

## 个人评注

本文是 Karpathy Software 3.0 理论的系统梳理。对 Tizer 的启示：OpenClaw 的 Agent 架构本身就是 Software 3.0 的实践——用 prompt 和 context window 驱动 Agent 编译知识。文中关于 Harness 工程的讨论直接映射到 OpenClaw 的 HITL 验证机制：每次发现 Agent 犯错都应该工程化为约束规则（如 Wiki Compiler 的去重检查、命名归一化），这正是 harness 思维在内容管线中的应用。
