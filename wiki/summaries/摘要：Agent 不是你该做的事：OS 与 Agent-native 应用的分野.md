## 一句话摘要

文章提出通用 Agent 更像 OS，而垂直团队真正的机会在于构建运行在其上的 Agent-native Application，用外部服务承接领域能力和业务状态。

## 关键洞察

- 垂类团队直接做 Agent OS，实际上是在跟通用模型和平台厂商打不对称战争。

- Prompt 或 Script 本身壁垒有限，真正的护城河来自领域状态、基础设施和规模经济。

- Agent 的两个核心物理约束是上下文容量和注意力带宽。

- 把领域推理和业务状态外置到应用层，本质上是在给 Agent 做认知卸载。
