## 一句话摘要

gnhf 是一个开源的 Coding Agent 隔夜编排器，让用户睡前设定目标后由 AI 编码助手按 Git 纪律持续迭代改进代码，次日收获干净的工作成果。

## 关键洞察

- **Git 纪律保障安全**：每次迭代成功即创建独立 commit，失败则 `git reset --hard` 回滚，连续 3 次失败自动中止，确保长时间无人值守运行的代码库完整性

- **Agent-agnostic 设计**：不绑定特定 Coding Agent，开箱支持 Claude Code、Codex、Rovo Dev、OpenCode、GitHub Copilot CLI、Pi 等，降低了切换成本

- **跨迭代共享记忆**：通过 [notes.md](http://notes.md/) 文件积累上下文，后续迭代能读取前序迭代的经验，实现隐式的长期记忆

- **Worktree 并行模式**：利用 git worktree 让多个 Agent 在同一仓库上并行运行不同任务，互不干扰

- **适用场景明确**：适合机械性、重复性工作（补测试、清理重复逻辑、降低复杂度），核心业务逻辑仍需人工审核

## 个人评注

gnhf 代表了 Coding Agent 工具链从「交互式辅助」向「无人值守自主执行」的演进方向。其 Git 纪律迭代循环模式与 OpenClaw 的 Harness 理念高度契合——都在探索如何为 Agent 建立安全边界和验证机制。对于 Tizer 的 HITL 工作流而言，gnhf 提供了一个可参考的「夜间 Agent」编排范式：白天人工审核方向，夜间 Agent 自动执行机械任务，形成高效的人机协作节奏。Worktree 并行模式也为多 Agent 协作场景提供了工程实现思路。
