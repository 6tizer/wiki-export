### 一句话摘要

Obsidian CLI 的意义不在于让 Agent 能读 Markdown，而在于让 Agent 能沿着知识库的链接网络理解上下文关系。

### 关键洞察

- 开启 CLI 后，Agent 拿到的不是单篇文件，而是整个链接拓扑的访问入口。

- backlinks 让 Agent 更容易发现隐藏在知识网络中的相关上下文。

- 对知识库型工作流来说，结构本身和内容一样重要。

- 早期命令的静默失败问题，提醒我们官方能力仍需配套技能层兜底。
