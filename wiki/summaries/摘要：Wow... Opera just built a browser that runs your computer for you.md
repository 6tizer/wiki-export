## 一句话摘要

Opera 推出 AI 原生 Agentic 浏览器 Opera Neon，将多个主流大模型集成于单一订阅中，通过内置自主 Agent 实现表单填写、邮件回复、深度研究等任务自动化，提出「谁控制浏览器，谁就控制工作流」的产品论点。

## 关键洞察

- **浏览器即 Agent**：Opera Neon 不是在浏览器上叠加聊天机器人，而是让浏览器本身成为能理解页面结构并自主执行操作的 AI Agent

- **多模型统一入口**：单一订阅整合 Claude Opus 4.6/4.7、GPT 5.x、Gemini 3.x、Grok 4.1、DeepSeek V3.2 等 18+ 模型，消除多 AI 应用切换成本

- **MCP 协议集成**：通过 MCP Connector Utility 打通浏览器与外部工具/服务，扩展 Agent 能力边界

- **Cards 任务卡片**：将重复性浏览器操作封装为一键触发卡片，实现个人工作流自动化

- **后台持续执行**：云端 Agent 在浏览器关闭后仍可继续工作，支持离线场景下的深度研究

## 个人评注

Opera Neon 代表了「浏览器作为 AI Agent 运行时」的产品方向。其 MCP Connector Utility 的集成思路与 OpenClaw 的工具编排理念有相通之处——都强调通过标准化协议扩展 Agent 的能力边界。Cards 功能的「将重复任务封装为可复用单元」也类似于 OpenClaw 的 Skill 抽象。值得关注的是其多模型统一调度策略，可以为 OpenClaw 的模型路由设计提供参考。不过整体内容偏推广性质，技术细节有限，置信度标记为 medium。
