## 一句话摘要

Exa 与 Coinbase 推进的 x402 支付模式，正在把网页搜索等 API 服务从“预先申请密钥”改造成“Agent 可按次自主付费调用”的机器支付基础设施。

## 关键洞察

- Exa 在无 API Key 请求下返回 402 支付信息，意味着 Agent 可以直接根据支付挑战完成搜索调用，不必再经过人工配置计费后台。

- 这条路径把 HTTP 请求、稳定币结算与工具调用串成了统一流程，使“按次购买能力”成为 Agent 原生工作流的一部分。

- 回复线程中，AgentPay 与 Orbis 被提及为已上线的 x402 生态案例，说明 x402 已开始从协议叙事走向工具市场与 API 网关落地。

- 讨论也暴露出该模式的现实争议：钱包管理、权限范围、成本透明度与大规模采用时机，仍是 Agent 支付体系必须解决的问题。

## 个人评注

这类“先发现工具、再按次付费调用”的能力，与 Tizer 面向 Agent 工作流的方向高度相关。若未来内容编译、研究检索或 OpenClaw 相关流程接入 x402 式支付轨道，很多原本依赖人工配置 API Key 的外部能力，都可以变成可预算控制、可动态购买的标准化步骤。这样既能降低接入摩擦，也更适合把外部工具真正编排进长期自动化管线。
