## 一句话摘要

这篇综述提出“外部化”是理解 LLM Agent 演进的统一框架：真正决定 Agent 可靠性的，不再只是更强模型，而是把记忆、技能、协议与执行环境系统性外置成可检索、可复用、可治理的认知基础设施。

## 关键洞察

- Agent 落地的核心瓶颈，不只是推理能力不足，而是连续性、一致性、协调性三类结构性错配。

- 记忆系统把“回忆”改写为“检索”，技能系统把“即兴生成”改写为“组合复用”。

- 协议把临时约定变成结构化契约，使工具、其他 Agent 与用户界面之间的协作更稳定、可移植。

- Harness 不只是运行容器，而是把执行流、权限、沙箱、可观测性和审批回路显式化的系统层。

- 评估 Agent 的关键问题，正在从“模型有多强”转向“哪些认知负担已经被成功外部化”。
