## 一句话摘要

这篇文章指出 GitHub 刷 Star 已形成成熟黑产，AI/LLM 项目是重灾区，而 GitHub Star 又被融资与榜单机制放大为关键外部信号，最终形成了“买星—上榜—融资”的恶性循环。

## 关键洞察

- CMU 团队基于 2019–2024 年 GitHub 数据识别出约 600 万颗疑似假 Star、超过 1.8 万个刷星仓库，说明问题已经系统化、规模化。

- StarScout 通过识别僵尸账号、同步批量点星等异常行为，为 GitHub 假 Star 提供了可规模化检测的方法。

- 假 Star 市场已经形成分层定价、账号农场、API 批量下单与“长期锁星”等完整服务链，说明这不是零散作弊，而是成熟产业。

- AI/LLM 项目在非恶意项目中属于刷星重灾区，部分项目甚至借虚假热度进入 GitHub Trending，说明推荐分发也会被操纵。

- 对投资人与使用者而言，Star 不再是可靠的单一质量指标，应该结合 last commit、更新频率、Issues、核心代码质量与 fork-to-Star 比例综合判断。
