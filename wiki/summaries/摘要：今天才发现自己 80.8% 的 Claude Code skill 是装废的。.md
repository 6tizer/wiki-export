## 一句话摘要

Claude Code Skill 的 description 写法直接决定其能否被 AI 主线程触发——「功能介绍型」描述会导致 Skill 形同虚设，应改为「Use when + NOT when」触发条件型格式。

## 关键洞察

- 80.8% 的 Claude Code Skill 处于「装废」状态，根本原因在于 description 写法不当

- 「功能介绍型」描述（如"用于 X / X workflow"）无法让 AI 主线程判断何时该调用该 Skill

- 「触发条件型」描述（"Use when … / NOT when …"）能让 AI 明确知道调用条件

- 一个含 52 个 Skill 的项目（codeburn）subagent 调用率仅 0.18%，修改 description 后预计显著提升

- 该洞察来源于 Tw93 的分享，属于实践经验总结

## 个人评注

这个发现对 OpenClaw 的 Skill 体系有直接参考价值——在设计 Skill 的 [SKILL.md](http://skill.md/) 和 description 时，应该采用「Use when + NOT when」格式来确保 Agent 能正确路由到对应的 Skill。这也属于 Harness 工程的范畴：如何通过声明式规则让 Agent 系统更可靠地调度子能力。
