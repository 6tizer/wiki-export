### 一句话摘要

MPP 把 Agent 支付从“人类代付的自动化补丁”推进成原生协议层能力，Stripe 则借 Tempo 抢占 Agentic Commerce 的基础设施入口。

### 关键洞察

- Session 机制把微支付聚合成持续授权语义，解决了 Agent 高频调用下的结算摩擦。

- Tempo 负责把稳定币链上结算隐藏在熟悉的 Stripe API 背后，降低开发者迁移成本。

- MPP 与 x402、AP2 并不是简单替代关系，而是分别占据协调层、执行层和信任层位置。

- Stripe 最大的护城河不是协议本身，而是现成的商户网络和信用卡入口。
