## 研究问题

**LLM 应用面临的安全威胁可以归纳为哪几个层次？不同层次的攻击手法、防御策略和实际影响有何异同？对 Agent 开发者而言，应如何构建多层防御体系？**

## 综合分析

### 一、LLM 安全威胁的三层模型

从 6 个交叉概念中，可以提炼出一个清晰的三层威胁模型：

| 威胁层 | 攻击目标 | 代表概念 | 攻击者类型 | 影响范围 |

| --- | --- | --- | --- | --- |

| **L1 模型层** | 模型行为与指令边界 | 提示注入、间接提示注入 | 恶意用户 / 第三方内容 | 单次会话 → 数据泄露 / 越权执行 |

| **L2 应用层** | 系统配置与知识产权 | System Prompt 泄露 | 竞争对手 / 安全研究者 | 产品机密暴露 / 提示词被复制 |

| **L3 平台层** | 账户存续与服务可用性 | 账号风控、Claude 账号风控 | 平台方（防御性） | 服务中断 / 账号封禁 |

| 补充层 | 目标 | 代表概念 | 定位 |

| --- | --- | --- | --- |

| **隐私保护层** | 训练数据隐私 | 联邦学习 | 非攻击性，是防御基础设施 |

### 二、提示注入攻击的双路径分析

提示注入和间接提示注入构成了 LLM 安全的最核心威胁面，两者的攻击路径截然不同：

| 维度 | 直接提示注入 | 间接提示注入 |

| --- | --- | --- |

| **攻击入口** | 用户对话输入 | 外部内容（网页、文档、API 返回） |

| **攻击者控制** | 主动、实时 | 被动、预埋 |

| **检测难度** | 中（可通过输入过滤） | 高（恶意指令隐藏在合法内容中） |

| **高危场景** | System Prompt 提取、权限绕过 | 联网浏览、工具调用、文件读取 |

| **防御关键** | 输入校验 + 提示隔离 | 内容沙箱 + 权限最小化 |

**关键洞察**：间接提示注入对高权限 Agent 的威胁远高于直接注入，因为 Agent 在联网浏览和工具调用时会主动摄入不可控的外部内容。这意味着 **Agent 越强大（权限越高、工具越多），攻击面越大**。

### 三、System Prompt 泄露的双面性

System Prompt 泄露与提示注入构成了有趣的攻防关系：

- **作为攻击**：泄露产品的系统提示词、工具定义和模型配置，暴露商业机密

- **作为研究**：让开发者学习商业级 AI 产品的提示结构、工作流设计和安全边界设计

- **提取路径**：提示注入 → 网络请求逆向 → 公开样本比对

这说明 **提示注入是 System Prompt 泄露的主要攻击向量**，两者形成了攻击链：先通过提示注入突破指令边界，再提取系统配置信息。

### 四、账号风控 — 被忽视的「服务可用性」威胁

账号风控和 Claude 账号风控揭示了一个常被安全讨论忽略的维度：**对 LLM 用户来说，最常见的「安全事件」不是被攻击，而是被平台封号**。

| 风控维度 | 通用账号风控 | Claude 账号风控 |

| --- | --- | --- |

| **检测信号** | IP、设备指纹、支付、行为频率 | IP 类型、浏览器环境、支付工具、使用行为 |

| **判断逻辑** | 多信号综合异常检测 | 环境一致性评估 |

| **核心原则** | 行为可信度 | 像一个真实、稳定、合规的用户 |

| **防御策略** | 环境一致性 > 单项技巧 | 静态住宅 IP + 指纹浏览器 + 合规支付 |

### 五、联邦学习 — 从训练阶段解决隐私问题

联邦学习处于威胁模型的最底层（基础设施层），它不直接防御攻击，而是从根本上减少数据暴露面：

- **核心机制**：数据不离开本地，只传递模型参数更新

- **与其他安全方案的互补性**：

  - 联邦学习解决「训练数据隐私」

  - 提示隔离/沙箱解决「推理时安全」

  - 账号风控解决「服务可用性」

- [**FLock.io**](http://flock.io/)** 的创新**：用区块链解决联邦学习的激励缺失和恶意节点验证问题

## 关键发现

> **💡** 

  1. **Agent 的安全悖论：能力越强，攻击面越大** — 间接提示注入的威胁与 Agent 的权限和工具数量正相关；高权限 Agent（如有文件系统和网络访问权限的 OpenClaw Agent）面临的间接注入风险远高于只做对话的 LLM

  1. **三层威胁之间存在级联效应** — 提示注入（L1）可以触发 System Prompt 泄露（L2），而环境异常可能触发账号风控（L3）；防御必须是多层的，单层防御无法阻止级联攻击

  1. **System Prompt 泄露的「价值反转」** — 对攻击者是安全漏洞，对开发者却是最有效的学习材料；134k Star 的提示词仓库说明社区已将泄露的 System Prompt 当作事实上的「设计模式库」

  1. **账号风控是 LLM 开发者最高频的实际安全问题** — 在讨论高级攻防之前，大多数开发者首先要解决的是「如何不被封号」这个基础问题；环境一致性比任何单项技巧都重要

  1. **联邦学习 + 区块链是去中心化 AI 安全的潜在范式** — [FLock.io](http://flock.io/) 的实践表明，联邦学习的两个核心痛点（激励不足、恶意节点）都可以通过链上机制解决，这为 Crypto × AI 安全打开了新方向
