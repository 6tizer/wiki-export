## 研究问题

[模型能力管线的三段博弈：训练塑形、推理压缩与评测度量如何在 Agent 时代形成相互约束的共演闭环](syntheses/模型能力管线的三段博弈：训练塑形、推理压缩与评测度量如何在 Agent 时代形成相互约束的共演闭环.md) 揭示了训练塑形、推理压缩与评测度量之间的非线性共演闭环。当我们将目光投向一个「远距离」标签——图像生成——时，发现了一个惊人的结构同构：扩散模型的「噪声调度→步数压缩→感知评测」管线与 LLM 的「训练塑形→推理压缩→基准度量」管线在数学结构、工程权衡和认识论困境上呈现精确对应。**这种同构是表层的类比巧合，还是揭示了所有生成式 AI 系统共享的「能力管线」结构性假说？**

## 综合分析

### 一、训练阶段的同构：噪声调度 ≈ 损失景观优化

| **维度** | **图像生成管线** | **LLM 能力管线** | **共享数学结构** |

| --- | --- | --- | --- |

| 核心训练范式 | 扩散模型学习从噪声中恢复信号，通过噪声调度表控制训练进程 | LLM 通过梯度下降在损失景观中寻找最优解，通过学习率调度控制收敛 | 受控的迭代优化 + 调度函数 |

| 架构决策的推理预编程 | 扩散步数、U-Net/DiT 架构在训练时就预设了推理时的计算预算 | MoE 稀疏架构在训练时预编程推理效率（如 Qwen3.5 的 397B/17B 设计） | 训练时的架构选择是推理时效率的「编译期优化」 |

| 特征学习 | Sparse Reference Attention 学习非对称注意力模式以降低参考长度计算复杂度 | Attention Residuals 让深层信息不被稀释以提升推理质量 | 注意力机制的定制化是训练-推理耦合的关键 |

| 闭环运动表征 | Avatar V 将运动信号同时作为预测目标和条件信号参与训练闭环 | On-Policy RL 将推理行为直接编码为训练目标 | 输出反馈为输入的递归训练范式 |

**关键发现**：扩散模型的闭环运动表征（motion token 同时作为预测目标和条件信号）与 LLM 的 On-Policy RL（推理轨迹直接作为训练数据）在结构上精确对应——两者都实现了**输出-输入递归**，即模型的输出被反馈为训练输入，训练与推理的边界在这里消失。

**外部验证**：TreeRL（ACL 2025，清华/Caltech）将 on-policy 树搜索直接嵌入 LLM 的 RL 训练循环，提供密集的过程奖励信号（[arXiv:2505.02865](https://aclanthology.org/2025.acl-long.604.pdf)）。Prime Intellect 的 "Recursive Language Models" 博文将递归训练称为「2026 年的范式」（[primeintellect.ai/blog/rlm](http://primeintellect.ai/blog/rlm)）。ICLR 2026 论文 "What happens when generative AI models train recursively on each others' outputs?" 进一步研究了递归训练的收敛性与同质化风险（[openreview.net](http://openreview.net/)）。

### 二、推理压缩的同构：步数蒸馏 ≈ 量化/蒸馏

图像生成和 LLM 推理都面临同一个核心挑战：如何在保持质量的前提下降低推理成本。两个领域的解法在工程结构上惊人一致。

| **压缩策略** | **图像生成** | **LLM 推理** | **共享权衡** |

| --- | --- | --- | --- |

| 蒸馏 | CFG+DMD 两阶段蒸馏实现 10x 加速（如 Avatar V），把多步扩散压缩为少步生成 | 从闭源大模型推理轨迹蒸馏到小模型（如 Qwen-Distilled），把长思维链压缩为短推理 | 质量—速度权衡由蒸馏策略调解 |

| 步数/Token 减少 | DDIM、Consistency Models 减少扩散步数 | Speculative Decoding、KV Cache 减少推理 Token 消耗 | 减少迭代次数而不显著损失质量 |

| 思考前置优化 | 思考模型重写——先用推理模型分析需求，再把重写后的 Prompt 交给生成模型 | Prompt Caching、上下文压缩——先结构化输入，再执行推理 | 输入质量优化 > 模型能力堆砌 |

**关键发现**：「思考模型重写」是图像生成领域独立发现的一种推理优化范式——先调用推理型模型将模糊创意转化为结构化规格说明，再交给生成模型执行。这与 LLM 领域的 Prompt Engineering 和上下文压缩在功能上精确同构，但图像生成领域在工程化程度上领先：它已经将这一步骤产品化为通用的、模型无关的标准前置组件。LLM 领域虽已出现 AutoPDL（2025）等自动 Prompt 优化工具，但整体仍偏碎片化，尚未形成图像领域那样的标准化管线。**两个领域的差距正在缩小，但图像领域的工程化思路仍值得 LLM Agent 工作流借鉴。**

**外部验证**：腾讯混元的 PromptEnhancer（CVPR 2026）正是这一范式的产品化实例——用 Chain-of-Thought 推理模型重写用户 prompt，完全解耦精修与生成，适配任意 T2I 模型（[arXiv:2509.04545](https://arxiv.org/abs/2509.04545)）。HunyuanVideo 也发布了专用的 PromptRewrite 模型用于视频生成（[huggingface.co/tencent/HunyuanVideo-PromptRewrite](http://huggingface.co/tencent/HunyuanVideo-PromptRewrite)）。LLM 侧的对应物包括 AutoPDL（[arXiv:2504.04365](https://arxiv.org/abs/2504.04365)）和 VisualPrompter（[arXiv:2506.23138](https://arxiv.org/html/2506.23138v1)）。

### 三、评测度量的同构：感知指标 ≈ 基准跑分

两个领域的评测体系面临完全相同的认识论困境：**度量的东西不是我们真正关心的东西。**

| **评测困境** | **图像生成** | **LLM** | **共享认识论问题** |

| --- | --- | --- | --- |

| 指标与感知的剪刀差 | FID/CLIP Score 高不代表人眼觉得好看；照片真实感测试中 GPT-Image-2 在人物细节上胜出，但指标未必反映 | SWE-bench 高分不代表实际编程能力强；基准跑分与真实 Agent 执行能力的剪刀差正在扩大 | 标准化指标度量的是「可测量的维度」而非「重要的维度」 |

| 评测作为训练信号的递归 | 当图像质量评分被用于 RL 训练奖励时，模型学会「讨好评审模型」而非提升真实质量 | LLM-as-judge 被用于 RL 奖励信号时，评测体系从外部观察者变成训练内部组件 | Goodhart 定律的跨域实例化 |

| 跨模态评测的不可比性 | 文本到图像、图像到视频、图像到 TikZ 代码——不同任务的评测标准完全异质 | 代码生成、对话、推理——不同能力维度的评测基准不可统一 | 多模态/多能力系统的评测碎片化 |

**SciTikZ 项目**揭示了评测困境的一个独特切面：科学图表转 TikZ 代码的评测要求不仅是「看起来像」（感知指标），还必须「代码可编译、可重现」（功能指标）。这在 LLM 领域的对应物是「不仅答案看起来对，还必须能通过测试用例」——两者都在试图从「感知相似度」走向「功能正确性」，但都面临功能正确性更难规模化评测的困难。

**外部验证**：

- **FID 指标失灵**：Google 的 *Rethinking FID*（CVPR 2024）系统论证了 FID 与人类评判矛盾、正态分布假设不成立、样本效率差等问题（[arXiv:2401.09603](https://arxiv.org/abs/2401.09603)）。

- **SWE-bench 失灵**：OpenAI 官方声明 SWE-bench Verified 已无法度量前沿编码能力，发现数据污染（[openai.com](http://openai.com/)）。arXiv 论文 *The SWE-Bench Illusion*（2025）进一步证实模型通过记忆化而非推理来解题，仅凭 issue 描述即可 76% 定位 bug 文件（[arXiv:2506.12286](https://arxiv.org/abs/2506.12286)）。METR 报告显示 LLM 代码「通过测试」的比例远高于「可被合并」的比例（[metr.org](http://metr.org/)）。

- **Goodhart 定律在 AI 评测中的实例化**：LMSYS/LMArena 排行榜争议是典型案例，模型优化排行榜分数而非真实能力（[blog.collinear.ai](http://blog.collinear.ai/)）。

### 四、三段闭环的跨域结构张力

图像生成管线与 LLM 能力管线在三段闭环的结构性矛盾上也呈现同构：

1. **训练→推理的预编程困境**：扩散模型的步数设计在训练时就预设了推理计算预算，正如 MoE 的 Expert 分片在训练时就预编程了推理调度方案。两者面临相同的刚性——训练时的承诺限制了推理时的灵活性。

1. **蒸馏—评测循环陷阱的跨域变体**：图像生成中的蒸馏模型在感知指标上接近教师模型，但可能丢失了细节质感。LLM 的蒸馏模型在基准上接近源模型，但可能丢失了深层推理能力。**两个领域的评测体系都无法可靠区分「真正学会」和「轨迹模仿」**——这是 Goodhart 定律在生成式 AI 中的普遍实例化。

  **作者实验验证（Epiplexity）**：Tizer 的 Epiplexity 实验（[github.com/6tizer/epiplexity](http://github.com/6tizer/epiplexity)）在文本蒸馏域微观层面复现了这一陷阱。该实验用 Llama 3.2 的 1B 和 3B 模型对同一文本计算困惑度（perplexity），定义两者的比值为「可学习性差距」（Epiplexity Gap），并在 193 篇 Wiki 文章上执行了排名扫描 + 多轮干预实验（5/29/38 篇，跨 8 个领域）。核心发现：用 3B 模型重写高困惑度文章后，1B 模型的 perplexity 平均降低 82%，但下游 QA 准确率反而从 4.5/5 降至 3.5/5——「蒸馏惖论」。机制分析显示两个因素：① **表征漂移**，3B 模型在重写时用自身训练先验替换了原始事实（类似于图像蒸馏丢失细节质感）；② **信号稀释**，文本膨胀 3–5 倍导致关键事实被填充文字淡化（类似于扩散步数减少时丢失高频信息）。Phase 4 进一步发现「域差异 > 策略差异」，即内容类型（医学/经济学 vs 叙事性文本）对蒸馏效果的影响远大于蒸馏策略的选择。

1. **评测时效性缺口**：图像生成的评测基准（FID 是 2017 年提出的）远远落后于模型能力的演进；LLM 的评测基准（MMLU 等）也面临饱和。两个领域都在经历评测体系与模型能力脱节的「度量真空期」。

### 五、从同构到合体：Diffusion LLM 的涌现

上述四节论证了扩散管线与 LLM 能力管线在结构上的对应关系。但最新的研究进展揭示了一个更强的信号：**两个领域不仅结构相似，而且正在架构层面直接融合。**

**Diffusion LLM（扩散语言模型）** 是这一融合趋势的核心载体：它们将扩散模型的迭代精修范式直接应用于文本生成，打破了自回归模型「逐 Token 生成」的顺序瓶颈。具体表现在三个方向：

1. **扩散架构做文本生成**：NeurIPS 2025 论文 *Accelerating Diffusion LLMs via Adaptive Parallel Decoding* 提出自适应并行解码（APD），通过将扩散模型与小型自回归模型的乘法混合，实现多 Token 并行生成，「反转了 Speculative Decoding 的标准设定」（[starai.cs.ucla.edu](http://starai.cs.ucla.edu/)）。

1. **扩散增强 LLM 推理**：LaDiR（*Latent Diffusion Enhances LLMs for Text Reasoning*）利用潜在扩散模型的迭代精修能力来增强文本推理，将扩散过程从图像域迁移到语义域（[arXiv:2510.04573](https://arxiv.org/html/2510.04573v6)）。

1. **Speculative Diffusion Decoding**：直接用预训练的扩散语言模型作为 draft 模型，在单次前向传播中生成多 Token draft，再由自回归模型验证——将扩散模型和自回归模型组装为协作管线（[arXiv:2408.05636](https://arxiv.org/html/2408.05636v1)）。

这一融合趋势的意义超越了「结构同构」的范畴：如果两个领域的管线仅仅是「长得像」，架构融合不会如此順畅。**它们能够在架构层面直接嵌合，说明底层的「迭代精修→压缩加速→质量评估」管线结构确实共享了相同的计算原语。** 这为第一节提出的结构性假说提供了一个来自工程实践（而非纯类比）的支撑角度。

**蒸馏同构的理论支撑**：NYU 的 *Why Knowledge Distillation Works in Generative Models: A Minimal Working Explanation*（2025）从理论层面证明，蒸馏在不同生成模型中表现出相同的 precision–recall 动态——质量（precision）提升伴随多样性（recall）下降（[arXiv:2505.13111](https://arxiv.org/html/2505.13111v1)）。Epiplexity 实验（[github.com/6tizer/epiplexity](http://github.com/6tizer/epiplexity)）在文本域实证了这一权衡：蒸馏后文本更通顺可预测（precision 提升，perplexity 降 82%），但领域特异性事实被模型先验替换（recall 下降，QA 准确率降低）。Phase 4 进一步表明 precision-recall 权衡的具体表现随内容域变化——术语密集型领域（医学/经济学）的蒸馏增益最大，叙事性文本则更易受信号稀释影响——暗示该权衡需要域参数化而非一刀切。

## 关键发现

1. **扩散管线与能力管线的同构不是表层类比，而是一个值得严肃验证的结构性假说**：训练阶段的「受控迭代优化 + 调度函数」、推理阶段的「蒸馏/步数压缩换取效率」、评测阶段的「感知指标 vs 功能正确性」——这三段结构在图像生成和 LLM 中独立演化但精确对应。这暗示可能存在一个更高层的抽象：**生成式系统倾向于经历「塑形→压缩→度量」的三段管线，且三段之间形成非线性共演闭环。** 目前该假说仅在扩散模型与 LLM 两个领域得到验证；要将其提升为跨域公理，还需在音频生成（如音乐/语音合成）、强化学习控制策略等更多领域检验其适用性，并识别可能的反例与适用边界。Diffusion LLM 的出现（见第五节）从架构融合的角度为该假说提供了额外支撑，但尚不构成充分证据。方法论上，Epiplexity Gap（不同规模模型对同一内容的困惑度比值，见第四节）提供了一种潜在的跨域验证工具：如果将类似的「模型尺寸感知差」指标迁移到图像/音频域（如用小扩散模型和大扩散模型对同一输入计算生成概率差），有望量化不同模态下蒸馏—评测耦合的强度，从而为假说提供跨域可比较的度量。

1. **「思考模型重写」是图像生成领域对 LLM Prompt Engineering 的工程化领先**：图像生成领域已将「先分析需求再生成」产品化为通用的、模型无关的标准前置组件（如 PromptEnhancer, CVPR 2026）。LLM 领域虽已出现 AutoPDL 等自动 Prompt 优化工具，但整体仍偏碎片化。两个领域的差距正在缩小，但图像领域的工程化管线思路——将 prompt 精修完全解耦为独立模块——仍是 LLM Agent 工作流可直接借鉴的方法论。

1. **Goodhart 定律是跨域的「评测原罪」，且在生成式 AI 中正在加速恶化**：当 FID/CLIP Score 和 SWE-bench/LLM-as-judge 同时被用作训练信号时，两个领域都在经历「优化指标 ≠ 提升能力」的系统性偏移。图像生成领域的照片真实感测试（用生活化场景横评而非标准 benchmark）为 LLM 评测提供了一个启示：**场景化评测可能比标准化基准更能反映真实能力。**

1. **闭环运动表征与 On-Policy RL 揭示了训练—推理边界消融的通用趋势**：两个领域都在走向「输出反馈为输入」的递归训练范式。Avatar V 的 motion token 递归与 LLM 的推理轨迹递归，预示着未来的训练不再是一个独立阶段，而是嵌入推理循环中的持续过程。这一趋势将重新定义「预训练」与「微调」的边界。

1. **图像生成的多任务评测碎片化（文本→图像、图像→代码、图像→视频）预示了 LLM 多能力评测的未来挑战**：SciTikZ 项目要求输出既「看起来对」又「代码可编译」的双重标准，在 LLM 领域的对应是「答案既要看起来合理，又要能通过测试用例」。随着 Agent 系统承担越来越多的多模态任务，评测碎片化将成为两个领域共同面临的核心挑战。

### 六、加速技术的双向流动：2025–2026 推理优化的跨域现状

本文第二节预测了图像与 LLM 领域在推理压缩上的同构关系。截至 2026 年初的实际进展表明，两个领域的加速技术不仅同构，而且已经**高度同步甚至双向流动**。以下按四条技术路线梳理。

**路线 1：一步生成蒸馏（图像 → 文本，已完成迁移）**

图像领域已将 50 步扩散压缩到 **1 步**，且蒸馏后学生模型 FID 超过教师模型（NVIDIA f-Distill，[research.nvidia.com](http://research.nvidia.com/)）。文本领域直接复制了这条路线：Inception Labs 的 **Mercury 2**（2026.02）在单 GPU 上达到 1009 tokens/秒，比同级自回归模型快 5-10 倍；**DLM-One**（[arXiv:2506.00290](https://arxiv.org/abs/2506.00290)）用分数蒸馏实现一步序列生成，推理提速约 500 倍。迁移时间差约 6-12 个月。

**路线 2：极端组合加速（双向同步）**

视频领域的 **TurboDiffusion**（清华，[arXiv:2512.16093](https://github.com/thu-ml/TurboDiffusion)）组合 SageAttention + 稀疏线性注意力 + 时间步蒸馏，在单张 RTX 5090 上实现 **100-200 倍**端到端加速（184 秒→1.9 秒）。文本领域已有等价组合：KV Cache 压缩（Trellis、LongFlow、量化 KV）+ Speculative Decoding（Apple QuantSpec、Yggdrasil 树状推测）+ 模型量化（FP4/FP8）。两个领域在此方向上齐头并进。

**路线 3：推理时搜索与验证（文本领域反过来领先）**

图像领域开始将经典搜索算法引入扩散采样：NeurIPS 2025 论文 *Inference-Time Scaling of Diffusion Models through Classical Search* 用爬山法、BFS/DFS 实现推理时 scaling（[diffusion-inference-scaling.github.io](http://diffusion-inference-scaling.github.io/)）；LATSEARCH 训练潜在奖励模型引导搜索方向。文本领域在此方向上走得更远：AB-MCTS（自适应分支蒙特卡洛树搜索，[arXiv:2503.04412](https://arxiv.org/abs/2503.04412)）、**Certaindex**（NeurIPS 2025，检测推理答案是否已稳定，稳定即退出）。Sebastian Raschka 2026 年初的综述将推理时 scaling 称为「当前最有效的提升 LLM 准确率的方式」。这是文本领域的搜索验证框架可以反向迁移到图像领域的方向。

**路线 4：自适应提前退出（平行发明，图像的理论保证领先）**

图像领域：ICLR 2026 论文 **TRACE** 分析扩散语言模型的训练动态，证明可以**安全地在一半步数时提前退出**，节省 11-68% 计算且无精度损失，提供了理论安全性保证（[openreview.net](http://openreview.net/)）。文本领域：**DEER**（Dynamic Early Exit in Reasoning）在推理模型上实现自适应退出，精度提升 0.9-4.8 分同时序列缩短 19-43%；**DiffAdapt** 根据问题难度自适应分配推理 Token。两个领域几乎同时发明了类似方案，但图像领域的 TRACE 提供的训练动态理论保证尚未被文本领域完全吸收。值得注意的是，Epiplexity Gap（见第四节）可能为自适应计算分配提供一种**输入层面的、模型外部的**前置判据（详见实验计划 A.5），与 DEER/DiffAdapt 的模型内部置信度判据形成互补。

**跨域流动总结**

| **技术路线** | **图像/视频领域** | **文本 AI 领域** | **流动方向** |

| --- | --- | --- | --- |

| 一步生成蒸馏 | f-Distill, DMD, Flow Map（已成熟） | Mercury 2, DLM-One, Gemini Diffusion | 图像 → 文本（6-12 月延迟） |

| 极端组合加速 | TurboDiffusion（100-200x） | KV 压缩 + Speculative + 量化 | 双向同步 |

| 推理时搜索 | Classical Search, LATSEARCH | AB-MCTS, Certaindex | 文本 → 图像（文本领先） |

| 自适应提前退出 | TRACE（理论保证）, LeADS | DEER, DiffAdapt, ADEPT | 平行发明；图像的理论保证可迁移到文本 |

这一现状对本文的「能力管线假说」具有双重意义：① 两个领域的加速技术高度同步本身就是假说的工程证据——如果管线结构不同，同一种加速技巧不可能在两个领域同时有效；② 流动方向已从「图像单向引领」变为「双向交叉迁移」，图像领域的优势在极端压缩和理论安全保证，文本领域的优势在搜索验证和自适应退出工程化。未来的加速突破更可能来自两个领域的交叉地带，而非单一方向的迁移。

## 后续实验计划

以下三个实验路径旨在验证本文提出的结构性假说，并将 Epiplexity 从文本域小实验拓展为跨模态度量框架。按优先级排序，每一步的输出为下一步提供基准。

### 实验 A（优先级最高）：Gap → 蒸馏有效性的相关性分析

**目标**：验证 Epiplexity Gap 能否预测蒸馏对下游任务的实际增益，建立文本域 baseline。

**方法**：

- 回到 Phase 4 的 38 篇文章数据，为每篇计算 Epiplexity Gap（PPL_1B / PPL_3B）

- 计算 Gap 与 Phase 4 蒸馏 QA 增益的相关系数（Spearman ρ）

- 按域分组（8 个领域），看 Gap 是否解释了「域差异 > 策略差异」现象

**预期结果**：

- 如果 Gap 与 QA 增益正相关：证明 Gap 可以作为「蒸馏值不值得做」的前置判断指标

- 如果无相关性：Gap 仅反映信息密度，不反映蒸馏可迁移性——也是有价值的反面发现

**执行要求**：使用 `epiplexity/results/checkpoints/` 中 Phase 4 的已有数据，运行相关性分析脚本，输出散点图 + ρ 值 + 分域箱线图。工作量约 30 分钟。

### 实验 A.5（A 的扩展）：Gap 作为自适应计算分配的前置判据

**目标**：验证 Epiplexity Gap 能否在推理开始前就预测最优计算预算，为自适应提前退出提供输入层面的外部判据。

**动机**：第六节路线 4 中 DEER、DiffAdapt 等自适应退出方法的判据都是**模型内部的**（看中间层表示是否稳定），必须在推理过程中动态判断。而 Epiplexity Gap 是**输入层面的、模型外部的**——只需对输入内容跑一次快速的小/大模型困惑度计算（几秒），即可在推理开始前确定计算预算。两者可以互补：Gap 做粗粒度预算分配，模型内部置信度做细粒度退出决策。

**方法**：

- 复用实验 A 的 38 篇文章 + Gap 数据

- 对每篇文章，用不同的推理 Token 预算（如 256/512/1024/2048 tokens）让模型回答相同问题

- 检验 Gap 是否预测了「达到最优 QA 分数所需的最小 Token 预算」

**预期结果**：

- 如果 Gap 小的文章在低 Token 预算时就达到最优，而 Gap 大的文章需要高预算：Gap 可作为计算分配的前置判据

- 反之则 Gap 仅反映信息密度，与推理计算量无关

**依赖**：实验 A 的 Gap 数据和相关性结果。仅在实验 A 确认 Gap 与蒸馏增益正相关时才执行。

### 实验 A.6（应用拓展）：Epiplexity Gap 的实际工作流应用

**目标**：将 Epiplexity Gap 从实验指标拓展为实际工作流中的实用工具。

**应用场景 1：Agent Skills 审计**

本地实验已将 Epiplexity Gap 应用于 Hermes Agent 的 100+ 技能文件，用于评估每个 Skill 的信息密度和可学习性。该数据和结论待补充。

**应用场景 2：Mem0 记忆系统审计**

对 Mem0 中存储的记忆条目批量计算 Epiplexity Gap，可实现三种应用：

- **记忆剪枝**：Gap 小的记忆（小模型和大模型都不困惑）= 常识性内容，即使不存在 Mem0 里模型也知道——可以安全清理，释放存储和检索开销

- **记忆质量审计**：两个模型都困惑的记忆（双高 perplexity）= 可能写得有歧义、信息不完整或太碎片化——标记为需要重写

- **检索优先级**：上下文窗口有限时，优先塞入 Gap 大的记忆（小模型特别困惑但大模型不困惑 = 真正有价值的领域知识），确保有限的上下文窗口用在最有价值的知识上

这些应用场景将 Epiplexity 从「蒸馏质量的度量工具」拓展为「知识系统的健康检查工具」，与本文讨论的评测困境直接关联——它们本质上都在回答同一个问题：「这条知识对谁有用、对谁多余？」

### 实验 B（优先级中）：图像域 Epiplexity Gap 迁移

**目标**：将 Epiplexity 的核心思路迁移到图像生成域，验证「能力管线」假说的跨模态适用性。

**方法**：

- 选取两个不同规模的扩散模型（如 Stable Diffusion 的小版本 vs 大版本，或 SDXL-Turbo vs SDXL-Base）

- 对同一组图像（跨多个域：人脸、风景、图表、抽象艺术）计算重建损失（或 ELBO）

- 定义「图像 Epiplexity Gap」= 小模型损失 / 大模型损失

- 然后对大模型做步数蒸馏（如 Consistency Distillation），评估蒸馏前后的 FID/LPIPS 变化

- 检验 Gap 是否预测哪类图像在蒸馏后质量损失最大

**预期结果**：

- 如果 Gap 大的图像类型（如细节密集的人脸）蒸馏后质量损失更大：跨域假说得到支撑

- 如果 Gap 与质量损失无关：图像域的蒸馏动态与文本域有本质差异，假说需要约束适用边界

**执行要求**：可通过 [fal.ai](http://fal.ai/) MCP 调用图像模型，或在本地 Apple Silicon + MLX 上跑小型扩散模型。需准备约 50–100 张跨域测试图像。工作量约 1–2 天。

**依赖**：实验 A 的结果作为文本域 baseline，用于跨域对比。

### 实验 C（优先级低）：结构性假说的验证边界与反例筛查

**目标**：定义「能力管线假说」的适用条件和反例边界，将其从模糊的跨域类比提升为可证伪的研究假说。

**方法**：

- 建立统一的验证框架：对每个待验证域，明确三段管线的对应物

| **域** | **塑形（训练）** | **压缩（推理）** | **度量（评测）** | **Epiplexity Gap 类比** |

| --- | --- | --- | --- | --- |

| 文本（已验证） | 语言模型预训练 | 蒸馏重写 | QA 准确率 | PPL_1B / PPL_3B |

| 图像（实验 B） | 扩散模型训练 | 步数蒸馏 | FID / LPIPS | 重建损失_small / 重建损失_large |

| 音频（待验证） | TTS/音乐模型训练 | 音频编码压缩 | MOS / FAD | 音频开放问题 |

| RL 控制（待验证） | 策略训练 | 策略蒸馏 | 任务成功率 | 小策略回报差 / 大策略回报差 |

| GAN（潜在反例） | 对抗训练 | 无显式迭代精修 | FID / IS | 可能不适用——三段管线的第一段缺失 |

- **成功标准**：如果 3 个以上域的 Gap 指标与蒸馏质量损失显著相关，假说升级为「跨域经验规律」

- **反例标准**：GAN 或其他无迭代精修阶段的生成模型，如果不符合三段管线，则明确假说的适用边界为「具有迭代精修训练范式的生成系统」

**执行要求**：等实验 A/B 有初步数据后再制定具体方案。当前仅需确认表格中的域列表和对应物是否合理。
