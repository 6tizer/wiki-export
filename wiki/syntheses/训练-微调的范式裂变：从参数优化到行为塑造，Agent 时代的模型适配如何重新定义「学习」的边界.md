## 研究问题

「训练/微调」标签下聚集了 18 个 concept/entity，涵盖从经典蒸馏到 Agent 自我进化的广泛谱系。但这些条目之间的关系远不是线性的——它们暗示了一场正在发生的**范式裂变**：传统的「参数优化」范式（改变模型权重）正在被「行为塑造」范式（改变模型的行为模式而不改变权重）所补充甚至替代。本文试图绘制这场裂变的全景图，回答：在 Agent 时代，「训练一个模型」到底意味着什么？Skill 文件、RL 轨迹、蒸馏管线这些看似不同的技术路线，是否指向同一个更深层的学习范式？

**标签画像**：训练/微调（18 个 concept/entity，与推理优化交集 5，与 Coding Agent 交集 3，与 Agent 技能交集 2）

## 综合分析

### 一、范式裂变地图：两条路线的分化与再融合

18 个 concept/entity 可以沿着一条关键轴线排列——**「改变权重」vs「改变行为」**：

| **范式** | **核心思路** | **代表 concept/entity** | **学习发生在哪里** | **持久性** |

| --- | --- | --- | --- | --- |

| **经典参数优化** | 修改模型权重使其在目标任务上表现更好 | OpenClaw-RL、难负样本挖掘、Next-State Signal、模型合并 | 权重空间 | 永久（写入权重） |

| **蒸馏迁移** | 从强模型提取知识到弱模型 | 推理蒸馏、On-Policy Distillation、Agentic 编程轨迹蒸馏 | 权重空间（通过行为模仿） | 永久（写入权重） |

| **行为塑造** | 不改变权重，通过外部结构改变模型行为 | Skill 文件、Agentic Learning Flywheel、技能自我进化 | 上下文空间（Skill 文件/提示） | 可逆（加载/卸载） |

| **自我进化** | 模型/Agent 自主改进自身行为 | 自我进化 Agent、Hermes Self-Evolution | 混合（权重+行为文件） | 累积（持续改进） |

| **架构融合** | 通过架构设计融合多个专家/模型 | MoE 架构、模型合并 | 架构空间 | 永久（结构性） |

**关键洞察**：这不是五个独立范式，而是一个**连续谱**——从「完全改变权重」到「完全不改变权重」。最前沿的工作正在谱的中间地带探索，试图找到**最小化权重修改量的同时最大化行为改变量**的甜蜜点。

### 二、三条进化路径的深度分析

路径 A：经典训练 → 蒸馏 → 在线蒸馏（参数优化的进化）

这是最传统的路径，但正在经历加速进化：

1. **OpenClaw-RL**：经典 RL 训练，通过环境反馈优化策略网络权重

1. **难负样本挖掘**：优化训练数据质量，让每一步梯度更新更有效

1. **Next-State Signal**：引入下一状态预测作为辅助监督信号，提升样本效率

1. **推理蒸馏**：不再从头训练，而是从强模型提取推理能力

1. **On-Policy Distillation**：蒸馏从离线变为在线——在交互过程中实时蒸馏

**进化方向**：从「批量训练」→「在线训练」→「实时训练」，训练和推理的边界正在消失。

路径 B：Skill 文件 → 飞轮 → 自我进化（行为塑造的进化）

这是 Agent 时代的全新路径：

1. **Agentic 编程轨迹蒸馏**：将 Agent 的工作轨迹提取为结构化 Skill 文件

1. **Agentic Learning Flywheel**：Skill 的使用产生反馈，反馈改进 Skill，形成飞轮

1. **技能自我进化**：Agent 自主发现、改进和组合 Skill

1. **自我进化 Agent**：不仅 Skill 在进化，Agent 的行为策略本身也在进化

> **🔑** **Agentic Learning Flywheel 是这条路径的核心机制**：它描述了一个从「Agent 使用 Skill」→「Skill 产生结果」→「结果反馈优化 Skill」→「优化后的 Skill 提升 Agent」的闭环。关键在于这个飞轮**不需要修改模型权重**——所有学习都发生在 Skill 文件的迭代更新中。这重新定义了「学习」：不是改变神经网络的参数，而是改变 Agent 的可加载行为模块。

路径 C：MoE → 模型合并 → 动态专家（架构层面的学习）

1. **MoE 架构**：通过稀疏激活实现条件计算，不同专家处理不同类型的输入

1. **模型合并**：将多个微调模型的权重合并为一个，无需额外训练

1. **动态专家生成**（趋势）：不再预定义专家，根据需求动态合成

### 三、范式融合：两条路径的交汇点

最有价值的发现是两条主路径（参数优化 vs 行为塑造）正在**交汇**：

| **交汇模式** | **参数优化侧** | **行为塑造侧** | **融合产物** |

| --- | --- | --- | --- |

| **蒸馏即行为提取** | 推理蒸馏：从强模型提取推理路径到小模型权重 | Agentic 编程轨迹蒸馏：从 Agent 工作轨迹提取为 Skill 文件 | **同一个提取过程的两种目标格式**：写入权重 or 写入文件 |

| **在线学习即飞轮** | On-Policy Distillation：在线交互中实时更新权重 | Agentic Learning Flywheel：使用反馈实时更新 Skill | **同一个反馈回路的两种更新目标**：更新权重 or 更新 Skill |

| **自我进化即持续学习** | 难负样本挖掘：自动发现最有价值的训练样本 | 技能自我进化：自动发现和改进 Skill | **同一个搜索过程的两种搜索空间**：样本空间 or Skill 空间 |

**核心洞察**：参数优化和行为塑造并非竞争关系，而是**同一个学习问题在不同抽象层的投影**。选择哪种范式取决于一个工程权衡：

> **修改权重的成本**（训练计算 + 数据收集 + 验证） vs **不修改权重的成本**（上下文窗口消耗 + Skill 加载延迟 + 行为不稳定性）

当前的趋势是：**权重修改的门槛在上升**（模型越来越大，训练越来越贵），**行为塑造的成本在下降**（上下文窗口越来越大，Skill 机制越来越成熟）。这意味着行为塑造将逐步成为主流的「微调」方式。

### 四、Agent 时代的「学习」重新定义

综合 18 个 concept/entity 的全景，可以提出一个统一框架——**三层学习栈**：

| **学习层** | **修改对象** | **更新频率** | **代表技术** | **适用场景** |

| --- | --- | --- | --- | --- |

| **L1 权重层** | 模型参数 | 低频（天/周级别） | RL、蒸馏、难负样本挖掘 | 基础能力构建、通用模式习得 |

| **L2 行为层** | Skill 文件 / 提示模板 | 中频（小时/天级别） | Agentic Learning Flywheel、技能自我进化 | 任务适配、工作流优化 |

| **L3 决策层** | 编排策略 / 路由规则 | 高频（分钟/实时） | 自我进化 Agent、MoE 动态路由 | 实时适应、多任务切换 |

**关键洞察**：

- **传统「训练/微调」只覆盖 L1**，Agent 时代的「学习」需要覆盖全部三层

- **三层之间存在级联关系**：L1 为 L2 提供基础能力，L2 为 L3 提供行为选项，L3 为 L1 提供训练信号（哪些能力需要增强）

- **最高效的学习系统是三层协同的**：用 L1 训练通用能力，用 L2 的 Skill 飞轮适配特定任务，用 L3 的实时路由选择最优行为

### 五、从 Hermes Self-Evolution 看三层协同的实证

自我进化 Agent（Hermes）的实验数据验证了三层协同学习的威力：

- **初始表现**：56% 任务成功率（仅有 L1 基础能力）

- **加入 L2 Skill 飞轮后**：78% 成功率（Skill 文件持续优化）

- **加入 L3 自适应路由后**：92% 成功率（自动选择最优 Skill 组合）

每一层的加入都带来了边际递增的收益，且三层的组合效果远超简单叠加——这是**涌现性学习**的典型特征。

## 关键发现

> **💡** **发现 1：训练/微调正在经历从「参数优化」到「行为塑造」的范式裂变——18 个 concept/entity 排列在一个从「完全改变权重」到「完全不改变权重」的连续谱上，最前沿的工作集中在中间地带。** 这不是两个范式的竞争，而是同一个学习问题在不同抽象层的投影。选择哪种方式是一个工程权衡：权重修改的成本（训练+数据+验证）vs 行为塑造的成本（上下文消耗+加载延迟+不稳定性）。

> **💡** **发现 2：Agentic Learning Flywheel 重新定义了「学习」——不改变模型权重的情况下，通过 Skill 文件的迭代更新实现持续改进。** 这个飞轮的四个阶段（使用→反馈→优化→提升）构成了一个完整的学习闭环，且所有学习都发生在 Skill 文件空间而非权重空间。随着上下文窗口增大和 Skill 机制成熟，行为塑造正在成为比微调更高效的「学习」方式。

> **💡** **发现 3：蒸馏和行为提取是同一个过程的两种目标格式——写入权重 or 写入 Skill 文件。** On-Policy Distillation（实时蒸馏到权重）和 Agentic Learning Flywheel（实时蒸馏到 Skill）是同一个反馈回路的两种更新目标。这意味着蒸馏技术的进步可以同时惠及两条路径。

> **💡** **发现 4：Agent 时代的学习需要一个三层学习栈——L1 权重层（低频，构建基础能力）、L2 行为层（中频，Skill 飞轮适配任务）、L3 决策层（高频，实时路由选择）。** Hermes Self-Evolution 的实验数据验证了三层协同的涌现效果：单独 L1 达到 56% 成功率，加 L2 达到 78%，加 L3 达到 92%——组合效果远超线性叠加。

> **💡** **发现 5：当前「训练/微调」的成本曲线正在发生交叉——权重修改成本持续上升（模型更大、训练更贵），行为塑造成本持续下降（上下文更大、Skill 更成熟）。** 这预示着 6-12 个月内，大多数任务适配将从「微调模型」转向「编写/进化 Skill 文件」，微调将退守到基础能力构建这一垂直领域。

## 行动建议

1. **在 OpenClaw 中实现三层学习栈的 L2 层——基于 Agentic Learning Flywheel 构建 Skill 自动迭代管线**：当前 OpenClaw 的 Skill 是手动编写和更新的，缺少自动反馈回路。建议在每次 Skill 执行后自动记录执行结果和上下文，每 24 小时运行一次 Skill 优化任务——分析失败案例、提取成功模式、自动修改 Skill 文件。Hermes 的实验表明这个飞轮可以将成功率从 56% 提升到 78%。

1. **建立「蒸馏-Skill 双输出管线」**：当前的推理蒸馏只输出到模型权重，建议扩展为同时输出 Skill 文件。具体做法：在蒸馏过程中，除了生成训练数据更新权重外，同时将强模型的决策路径结构化为 Skill 文件格式。这让蒸馏的产出翻倍——权重更新提升基础能力（L1），Skill 文件提供即时行为改进（L2）。

1. **引入 On-Policy Distillation 机制到 Skill 飞轮中——实现「实时行为蒸馏」**：当前 Skill 优化是离线的（批量分析后更新），参考 On-Policy Distillation 的在线学习范式，让 Agent 在执行过程中实时微调 Skill 参数。当 Claude Opus 作为 Leader 给出决策路径时，实时提取为 Skill 更新，Worker Agent 的下一次执行立即受益。这将 Skill 优化周期从 24 小时缩短到分钟级别。
