## 研究问题

图像生成、AI 设计和多模态架构三个领域各自快速演进，但它们之间的交叉地带正在产生只有同时审视三条边才能发现的涌现动力学。图像生成提供「生成能力」，AI 设计提供「约束语言」，多模态提供「理解底座」——这三者如何在相互约束与促进中形成共演闭环？统一模型架构的崛起是否正在改写这个三角关系的底层逻辑？

## 综合分析

### 一、三角模型：生成—约束—理解的共演回路

三个标签的交叉地带揭示了一个核心共演回路：

- **图像生成 → AI 设计**：生成能力的边界定义了设计语言的可表达空间

- **AI 设计 → 图像生成**：设计约束反向塑造生成模型的引导机制

- **多模态 → 图像生成**：理解能力决定生成的语义连贯性

- **图像生成 → 多模态**：生成任务推动多模态架构的统一表征研究

- **AI 设计 → 多模态**：设计意图要求跨模态一致性

- **多模态 → AI 设计**：跨模态理解为设计系统提供语义基底

### 二、边 A：图像生成 × AI 设计——结构化约束的崛起

图像生成与 AI 设计的交叉最核心的演进是：**从自然语言描述走向结构化设计约束**。

| **约束范式** | **技术表现** | **代表案例** | **设计控制粒度** |

| --- | --- | --- | --- |

| 自然语言 Prompt | 文本描述 | 通用提示词 | 粗粒度，不可控 |

| 结构化 Prompt | [Untitled](concepts/JSON Prompt.md) | promptsref 的 image-to-prompt | 字段级可编辑 |

| 像素级约束 | [Untitled](concepts/精确像素比例控制.md) | 明确像素尺寸写入 Prompt | 版式级精确 |

| 品牌级约束 | [Untitled](entities/Canva Design Model.md) | 品牌元素、字体、配色理解 | 设计系统级 |

[JSON Prompt](concepts/JSON Prompt.md) 的出现是一个关键拐点：它把图像描述拆成主体、光线、镜头、氛围等结构化字段，让提示词从「语义描述」变成「可编辑规格」。而 [promptsref](entities/promptsref.md) 进一步将其产品化，提供了从图片反向提取结构化 Prompt 的能力——这本质上是**设计语言和生成语言之间的双向翻译器**。

[Canva Design Model](entities/Canva Design Model.md) 则代表了更高级的融合：模型不仅生成图像，还理解图片结构、文本、字体、配色和品牌元素，生成可编辑的设计输出。这意味着**设计约束不再是外加于生成模型的后处理，而是嵌入模型的内在能力**。

### 三、边 B：图像生成 × 多模态——统一表征的范式革命

图像生成与多模态的交叉地带正在经历一场范式革命：**理解与生成的统一**。

[多模态理解生成统一](concepts/多模态理解生成统一.md) 揭示了当前业界的核心分裂：

- **分离派**：理解和生成分属不同模型（如 GPT-5.5 看图 + GPT Image 2 生图）

- **统一派**：单一模型同时具备理解和生成（如 SenseNova U1 基于 NEO-Unify 架构）

统一模型的核心优势在于：同一语境下通盘判断文本与视觉的配合，生成的图像与上下文内容语义连贯。其技术关键在于构建统一表征空间，让视觉信息无需先「翻译」为文本 Token 再参与推理。

[Vision Transformer](concepts/Vision Transformer.md) 作为多模态视觉理解的基座，已经展示了这种统一的可能性：通过对图像块序列建模，它把视觉任务纳入了 Transformer 的统一框架，而且本地实时视觉推理已变得可行。

[闭环运动表征](concepts/闭环运动表征.md) 则展示了统一的另一个维度：把「怎么动」视为独立于静态外观的身份信号，通过预测 motion token 与注入 reference motion token 形成训练闭环。这是**在统一表征内部进一步解耦外观表征和运动表征**的尝试。

### 四、边 C：AI 设计 × 多模态——设计意图的跨模态统一

AI 设计与多模态的交叉点是：**设计意图需要跨越模态边界保持一致**。

[Canva Design Model](entities/Canva Design Model.md) 明确提出目标是「跨照片、插画、文本、视频与音频统一表达设计意图」。这不是多个单模态模型的拼接，而是设计语义在统一表征空间中的原生流动。

[静态概念图动态化](concepts/静态概念图动态化.md) 是这一逻辑的典型应用：先用图像生成产出关键场景，再用视频生成补上动作、转场与 UI 反馈。关键不只是单镜头动效，而是多个场景之间的设计语言一致性。

### 五、三角中心：产品形态分化的三种路径

三条边的交汇在产品层形成了三种分化路径：

**路径 A：设计工具型**（设计约束主导）

Canva Design Model、[Font Generator](concepts/Font Generator.md)、[SVG Logo 生成](concepts/SVG Logo 生成.md)、[Logo Generator](entities/Logo Generator.md) 代表的是「品牌设计工坊」路线：生成能力被品牌规范、字体体系、设计系统所约束，输出的是可编辑设计稿而非纯图片。

**路径 B：创意生成型**（生成能力主导）

[本地模型推理](concepts/本地模型推理.md) 和 Open Generative AI 代表的是「创作者自由画布」路线：用本地推理压低边际成本，追求最大的生成自由度。

**路径 C：统一架构型**（多模态统一主导）

[多模态理解生成统一](concepts/多模态理解生成统一.md) 和 SenseNova U1 代表的是「统一表征架构」路线：模型同时看懂和画出，理解与生成共享同一套内部表征。

| **维度** | **设计工具型** | **创意生成型** | **统一架构型** |

| --- | --- | --- | --- |

| 约束源 | 品牌规范 + 设计系统 | 用户创意 + 美学直觉 | 语义上下文 + 统一表征 |

| 多模态依赖 | 强（理解设计元素） | 弱（主要生成） | 最强（理解生成一体） |

| 输出形态 | 可编辑设计稿 | 终态图像/视频 | 多模态原生内容 |

| 商业模式 | 品牌订阅/SaaS | 免费+本地/开源 | 平台级 API |

### 六、核心张力：专精化 vs 统一化的架构博弈

三角关系中最深层的张力在于：**专精化与统一化的博弈**。当前的趋势是：**统一化在架构层胜出，但专精化在产品层仍有明确优势**。SenseNova U1 等统一架构在理解生成一致性上更强，但 Canva Design Model 等专精模型在品牌设计场景的精确度上更强。

## 关键发现

1. **结构化提示词是设计语言和生成语言之间的「双向翻译器」**：JSON Prompt 和 promptsref 的 image-to-prompt 能力共同构成了「设计意图 ↔ 生成指令」的双向转换通道。这不仅是提示词工程的进化，而是 AI 设计与图像生成之间形成闭环反馈的关键基础设施。

1. **统一表征的真正价值不在于「理解+生成」，而在于「设计一致性」**：多模态理解生成统一的本质价值不只是「一个模型做两件事」，而是在统一表征空间中，生成的图像能够与上下文的设计语义保持一致——这正是 AI 设计最需要的能力。

1. **运动表征的解耦预示着设计原子的重新定义**：闭环运动表征把「怎么动」从外观中解耦出来，这意味着未来的 AI 设计系统可能不再以「静态图像」为设计原子，而是以「外观+运动+交互」的多维表征为设计单元。

1. **本地推理打开了「设计试错」的经济学新空间**：本地模型推理的价值不只是离线和隐私，更关键的是它把生成的边际成本压到接近零，让设计试错过程可以真正「无限迭代」——这对 AI 设计的工作流有结构性影响。

1. **Canva 的自研模型战略揭示了三角融合的终局形态**：Canva Design Model 同时对标了三个维度：基于结构化设计数据训练（AI 设计）、生成可编辑设计稿（图像生成）、跨模态统一表达设计意图（多模态）。这是三角关系融合为单一产品的最立体案例。

## 行动建议

1. **为内容管线引入 JSON Prompt 作为标准化的视觉生成接口**：当前 Tizer 的内容管线在生成图像时依赖自然语言描述，建议引入 JSON Prompt 格式作为中间表示层，让设计意图的传递更可控、可复现。

1. **密切跟踪统一表征架构的产品化进展，评估替换当前分离式生成管线的时机**：多模态理解生成统一架构可能是下一个里程碑级范式，建议持续收录相关模型的评测数据和产品案例，为切换时机做准备。

1. **探索「设计规范 → 结构化 Prompt → 批量生成」的自动化链路**：结合 design-md-chrome 的设计规范提取与 JSON Prompt 的结构化描述，可以构建从「参考网站 → 设计规范 → 结构化 Prompt → 风格一致的批量视觉内容」的全自动链路。
