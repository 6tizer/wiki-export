## 定义

Canonical Design 是由 3uild 提出的品牌基础设施方法论，核心主张：在 Agent 时代，品牌的首次印象不再是视觉（logo、配色、标题），而是 AI Agent 对品牌信息的解析结果。品牌需要为「两个读者」——人类和机器——同时设计内容。

传统品牌设计针对人类视觉感知优化（排版、层次、留白），但如今越来越多的品牌发现发生在 AI 问答场景：用户向 Agent 提问，Agent 基于能抓取到的信息回答，品牌的叙述权由此旁落。Canonical Design 要求品牌创建「规范记录」（canonical record）——签名的、结构化的、发布在永久 URL 上的权威数据，让 Agent 读取而非猜测。

## 关键要点

- **双读者原则**：同一品牌制品需同时服务人类（视觉感知、情感共鸣）和机器（结构化解析、事实提取）

- **规范记录**：品牌应在自己控制的 URL 发布结构化、签名的权威数据，作为 Agent 的信息源

- **表面适配**：相同内容在不同 Agent 输出表面（Slack、Notion、邮件等）需正确渲染，Slack 的 Markdown 渲染差而 Notion 表现优秀

- 传统品牌实践（视觉识别、艺术指导、网页设计）是为单一读者建造的，现在需要重构
