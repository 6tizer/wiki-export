## 定义

Software 3.0 是 Andrej Karpathy 在 AI Ascent 2026 上提出的概念框架，用于描述软件开发的第三个范式——以大语言模型为核心的编程模式。与 Software 1.0（人类手写代码）和 Software 2.0（神经网络权重通过训练学习）不同，Software 3.0 的「代码」是自然语言 prompt，而 Agent 则充当这些 prompt 的安装器和执行器。

**别名**：Software Three Point Zero

## 关键要点

- Software 1.0 = 人类写代码；Software 2.0 = 训练数据写权重；Software 3.0 = 自然语言写 prompt

- Agent 是 Software 3.0 的「安装器」（installer），负责将自然语言意图翻译成可执行流程

- 与传统软件不同，Software 3.0 的技能边界是「参差不齐的」（jagged）——模型在某些任务上超人，在另一些上却出人意料地弱

- 可验证性（verifiability）是决定 Software 3.0 适用场景的关键维度
