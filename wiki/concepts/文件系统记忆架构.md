## 定义

文件系统记忆架构（Filesystem-based Memory Architecture）是一种 Agent 记忆持久化方案，Agent 直接使用 bash 和代码执行工具在文件系统中读写记忆条目，无需额外的检索栈或向量数据库。记忆以文件形式存储，天然支持版本控制、审计和回滚。

## 关键要点

- Agent 复用已有的代码执行工具（bash、文件读写）来管理记忆，降低架构复杂度

- 文件存储使记忆可审计、可回滚，满足企业合规需求

- 记忆按 session + agent 维度隔离，跨 Agent 共享需严格权限控制

- 相比向量数据库方案，文件系统方案更透明但检索能力受限

- 需要关注记忆膨胀和过时条目清理（provenance 问题）
