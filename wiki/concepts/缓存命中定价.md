**定义**：缓存命中定价（Cache Hit Pricing）是 LLM API 供应商针对输入 token 中命中 KV Cache 的部分给予大幅折扣的定价策略。由于重复/相似的上下文前缀可以复用已计算的 KV Cache，推理成本显著低于未命中的输入 token，供应商据此差异化定价。

**关键要点**

- DeepSeek V4 的缓存命中输入价格低至未命中价格的 1/10（折上折再打 1 折）

- 在 Agent 编程等长上下文、多轮对话场景中，缓存命中率可达 91%–96%，实际成本节省可达 83%

- 该策略使得百万级上下文窗口的实际使用成本大幅降低，推动"百万上下文普惠时代"

- 缓存命中定价是永久性的，非限时优惠

**与 Tizer 工作流的关联**

- OpenClaw Agent 的多轮编程任务天然具有高缓存命中率，直接受益于此定价模式

- 可据此优化 Agent 的 prompt 设计，提高前缀复用率以进一步降低成本
