## 定义

记忆技术债（Memory Technical Debt）是指 AI Agent 在长期运行过程中，其记忆系统积累的**过时、矛盾、冗余信息**，导致 Agent 决策质量下降的现象。类比软件工程中的技术债概念，记忆技术债如果不定期清理，会让 Agent「越记越笨」。

## 常见表现

- **决策矛盾**：旧记忆中的决策与新决策冲突，Agent 无法判断哪个优先

- **时间过时**：记忆中包含「下周」「明天」等相对日期，随时间推移失去意义

- **项目残留**：已完结项目的笔记仍占据记忆空间，干扰当前任务

- **重复冗余**：同一信息被多次记录，浪费 context 预算

## 要点

- 这是一个被广泛低估的问题——大多数 Agent 记忆方案只关注「如何记住更多」，忽略了「如何遗忘和整理」

- Claude Code 的 Auto Dream 功能专门解决记忆技术债问题

- 宝可梦实验案例：Sonnet 3.5 累积 31 个记忆文件反而表现更差，说明记忆数量不等于记忆质量

- 定期清理记忆（类似 Wiki Lint）是维持 Agent 长期表现的关键基础设施
