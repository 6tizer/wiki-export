## 定义

读改比是衡量编程 Agent 在执行任务时“读取上下文”和“动手修改”之间比例关系的操作性指标，用来判断 Agent 是否先理解再行动。

## 关键要点

- 较高的读改比通常意味着 Agent 会先检查代码、文件和上下文，再进入修改阶段

- 较低的读改比可能意味着 Agent 过早动手，容易出现误改和漏看上下文的问题

- 这个指标适合拿来观察 Claude Code 一类工具是否出现了“看一眼就改”的降质现象
