## 定义

异步事件驱动编排是一种多 Agent 协作模式。父 Agent 在派生子 Agent 后立即继续处理其他事务，子 Agent 完成时再通过事件推送结果回流，从而形成非阻塞的协同闭环。

## 关键要点

- 父 Agent 不必同步等待子任务结束，可以并行管理更多运行中的子任务

- 结果通过 announce、消息推送或事件流回传，减少轮询开销

- 更适合复杂工作流、长耗时任务和需要动态重规划的系统

- 代价是状态管理、上下文膨胀和通信控制复杂度显著上升
