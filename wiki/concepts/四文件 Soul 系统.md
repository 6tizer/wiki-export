## 定义

四文件 Soul 系统是一种用多个纯文本 Markdown 文件拆分定义 Agent 身份的配置范式，通常把人格、偏好、行为节奏与长期设定分别写入独立文件，便于阅读、修改与版本管理。

## 关键要点

- 相比把记忆或人格埋进数据库，这种方式更透明、更适合 Git 版本控制。

- 它强调“身份是文件”，而不是隐藏在不可见状态里的黑箱参数。

- 在 Mercury 的表述里，这四个文件分别是 `soul.md`、`persona.md`、`taste.md`、`heartbeat.md`。

- 这种模式适合需要长期协作、反复调优人格和输出风格的 Agent 系统。
