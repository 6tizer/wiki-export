## 定义

代码编排是指只暴露少量通用能力入口，由 Agent 通过脚本或代码把搜索、执行、过滤、组合等步骤串起来完成复杂任务的编排方式。

## 关键要点

- 与为每个 API 端点单独暴露一个工具相比，代码编排更薄、更通用，也更容易跨场景复用。

- 它强调“能力接口尽量少，执行逻辑尽量放在代码里”，因此能降低工具 schema 的臃肿程度。

- 在云端 MCP 场景里，代码编排相当于把 CLI 的组合哲学迁移到远程协议层。

- 它适合 API 面巨大、任务路径多变的系统，例如需要先搜索再执行的服务型工具集。
