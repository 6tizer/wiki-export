## 定义

Memory Lock-in 指 Agent 的历史状态、用户偏好、长期记忆与压缩产物被绑定到特定平台或模型提供商，导致一旦迁移 Harness、模型或部署平台，就无法无损继承既有经验的锁定效应。

## 关键要点

- 真正难迁移的往往不是模型本身，而是已经积累下来的状态与记忆

- Stateful API、server-side compaction 与黑箱长期记忆都会放大这种锁定

- 一旦记忆与 Harness 被整体收进托管 API，开发者会同时失去可见性、可控性与可迁移性
