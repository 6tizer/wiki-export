## 定义

上下文污染（Context Pollution）指在多轮对话或多 Agent 协作场景中，旧的、过时的或错误的上下文信息残留在会话中，导致后续生成结果受到干扰并产生错误。

## 关键要点

- 在长对话中，即使用户已明确纠正了某个信息，AI 可能仍会沿用旧上下文中的错误版本

- 跨 session 传递指令时尤其容易发生：分多次发送的指令片段可能导致 Agent 保留了已被否定的早期设定

- 上下文污染与上下文溢出不同：前者是「质」的问题（错误信息残留），后者是「量」的问题（信息量超出窗口容量）

- 解决策略：一次性发送完整指令、创建新 session 重新开始、或在关键节点显式清除旧设定

## 典型场景

- 多 Agent 协作中，A Agent 纠正了角色名称，但 B Agent 仍使用旧名称

- 长对话中反复修正同一参数，模型在不同回复中混用新旧值

- 跨平台传递上下文时，JSON 或 prompt 中遗留了过时字段
