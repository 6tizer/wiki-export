## 定义

持久状态是指 Agent 在一次会话结束、任务中断或服务 scale-to-zero 之后，仍能保留工作现场的能力，包括文件系统、中间产物、任务上下文与必要记忆，从而支持真正的长时运行工作流。

## 关键要点

- **不是上下文窗口的替代品**：它解决的是跨会话延续执行，而不是单次对话里的 token 记忆

- **是长任务前提**：没有持久状态，编码、调研、运维这类多阶段任务很难稳定交付

- **支撑可恢复性**：任务中断后可以从已有状态继续，而不是每次从零开始

- **强化可审计性**：状态保留让回放、追责与治理更容易落地

## 代表场景

- 夜间自动重构代码的 Coding Agent

- 持续数小时的信息检索与整理任务

- 需要跨步骤 handoff 的多 Agent 工作流
