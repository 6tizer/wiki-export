## 定义

辅助模型路由（Auxiliary Model Routing）是一种在 AI Agent 系统中为不同子任务分配不同 LLM 的策略。主推理任务使用高能力模型（如 Claude Opus），而上下文压缩、会话摘要、标题生成、视觉识别等辅助任务则使用更轻量、更便宜的模型（如 Haiku），从而在性能和成本之间取得平衡。

## 关键要点

- Hermes Agent 允许用户为每类辅助任务单独指定模型

- 典型分工：Opus 做主推理，Haiku 做压缩，小模型做标题生成

- 核心价值：避免用顶级模型处理低复杂度任务，显著降低 token 成本

- 在 Hermes 中通过 `hermes model` 配置界面的 auxiliary 屏幕统一管理

## 与其他概念的关系

- 与模型部署（serving）相关：需要同时接入多个模型端点

- 与成本优化直接相关：是降低 Agent 运行成本的关键架构决策

- 与多 Agent 协作不同：辅助模型路由是单 Agent 内部的模型分工，而非多个独立 Agent 间的协作
