## 定义

Agent-native Application 是运行在 Agent 交互范式之上的垂直应用层，不直接竞争通用 Agent OS，而是把领域能力外化为可被 Agent 调用的服务。

## 核心要点

- 护城河来自领域状态、基础设施和规模经济，而不是单个 Prompt 或脚本

- 通过外部服务承接复杂业务逻辑，减轻 Agent 上下文与注意力负担

- 更像“跑在 Agent 上的应用”，而不是“另一个 Agent OS”
