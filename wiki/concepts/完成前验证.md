## 定义

完成前验证（Verification Before Completion）是 Claude Code 工作流中的一条核心规则：未运行验证命令并读取输出，就不得声称任务完成。旨在消灭 AI 编程助手最常见的失败模式——「没检查就说应该可以了」。

## 关键要点

- **铁律**：在宣称完成前，必须运行相关验证命令（如 `npm test`、类型检查、lint）并读取输出

- **消灭的失败模式**：AI 编码助手常在修改后直接说「应该好了」而不验证

- **实现方式**：作为 Claude Code 工作流的 Skill 实现（verification-before-completion/[SKILL.md](http://skill.md/)）

- **哲学**：「验证 > 感觉」——跑一遍 `npm test` 比上线坏构建便宜

- **与 Hook 配合**：v2 中可通过 PostToolUse Hook 自动触发验证检查
