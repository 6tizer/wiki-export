## 定义

**反蒸馏**（Anti-Distillation）是针对人格蒸馏/知识蒸馏的对抗性保护技术。代表项目：`leilei926524-tech/anti-distill`（1700 Stars）。

核心思路：**清洗你被迫写的 Skill 文件**——让 skill 文件看起来完整，但核心知识保留在本人手中，防止他人通过蒸馏获取你的核心知识产权和思维模式。

## 使用场景

- 员工被要求写 colleague-skill 时的自我保护

- 防止雇主或团队通过 AI 蒸馏方式提取员工核心知识

- 在贡献蒸馏内容时保留个人独特竞争力

## 技术原理

- 注入噪声信息以混淆蒸馏结果

- 保留表面风格特征但隐去深层决策逻辑

- 类似水印（watermarking）思路在人格层面的应用

- 在 Claude Code 场景中，常见实现包括伪造工具注入，以及将工具调用细节摘要为类别统计，降低输出被直接蒸馏利用的价值

## 厂商级实现：Claude Code 案例

作为大模型厂商在产品中内置反蒸馏（防止竞争对手通过抓取 API 输出进行蒸馏训练）的典型案例，Claude Code 源码泄露后其反蒸馏设计被外界发现：

1. **伪造工具注入**：在模型输出流中注入虚假、不可用的工具调用，污染被抓取的数据，使其难以被有效用于训练

1. **输出类别摘要**：将工具调用的具体细节摘要为类别统计（如搜索、读取、写入、命令等），让外部难以还原 Claude 的完整推理链路

1. **Feature Flag 开关**：在 `services/api/claude.ts` 中通过 feature flag 控制：`anti_distillation: ['fake_tools']`
