## 定义

x402 是基于 HTTP 402 状态码（「付款必须」）设计的链上支付标准协议，由 Coinbase 提出，让 AI Agent 可以在发起 HTTP 请求时自动完成付款，无需人工签名。

## 技术特点

- 基于 HTTP 402 状态码：当 Agent 访问需付费的资源时，服务器返回 402 并提供支付信息

- 偏向无状态、按次结算，适合简单直接的链上微支付场景

- 使用 Base 链上的 USDC 结算，交易确认时间约 200 毫秒

- 手续费不足 $0.001

- AINFT 将这一协议引入 TRON 生态

## 意义

x402 让 AI Agent 真正拥有「自主支付能力」：不只是执行任务，而且能在授权金额范围内自动完成特定条件下的链上付款，全程无需人工介入。是实现「Agent 作为链上经济实体」愿景的关键基础设施。

## 已知风险

- 高权限操作：Agent 在授权范围内自主执行不可逆交易，一旦受到提示注入攻击风险高

- 现处早期阶段，标准化程度有限
