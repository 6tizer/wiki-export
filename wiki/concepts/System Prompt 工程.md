## 定义

System Prompt 工程是把模型的角色设定、行为边界、输出格式与问题求解流程系统化设计的实践，用来稳定提升 Agent 在真实任务中的表现。

## 关键要点

- 它不只是“写一段更长的提示词”，而是设计一套可迁移、可维护的行为约束系统

- 在多模型切换、子 Agent 调度与复杂任务执行中，常常比换模型本身更重要

- 与上下文注入、角色定义、任务验证机制结合后，能够明显拉齐不同模型的执行质量
